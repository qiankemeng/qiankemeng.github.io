# CMS maintenance

This project uses Decap CMS for local visual editing of blog content.

## Current maintained mode

The maintained default is local development mode:

```yaml
backend:
  name: git-gateway
  branch: main

local_backend: true
```

This avoids relying on Netlify Identity for day-to-day maintenance and keeps content changes inside the local git workflow.

## Start CMS locally

Run two terminals:

Terminal 1:

```bash
npx decap-server
```

Expected signal:

```text
Decap Server listening on port 8081
```

Terminal 2:

```bash
npm run dev
```

Then open:

```text
http://localhost:3000/admin
```

## Important files

- `public/admin/index.html` — CMS entry
- `public/admin/config.yml` — active CMS config
- `public/admin/config-local.yml` — local-mode reference config
- `public/admin/config-production.yml` — historical production/OAuth reference
- `public/admin/config-simple.yml` — minimal test config
- `src/content/blog/posts/` — generated/edited Markdown posts
- `public/images/blog/` — uploaded blog images

## Content collections

The intended categories are:

- `research` — Qianke's own research articles and paper writeups
- `daily-papers` — AI-curated arXiv papers, normally generated by the paper agent
- `tutorials` — technical tutorials
- `notes` — study/research notes
- `others` — miscellaneous posts

If a CMS collection points to a directory that does not exist yet, create the directory with a `.gitkeep` only when that collection is actively used.

## Troubleshooting

### `/admin/` blank page

Check browser console first.

Common causes:

- `config.yml` failed to load
- YAML syntax error
- CMS backend mismatch
- Decap CDN/network issue
- local backend server not running

Quick local checks:

```bash
npm run dev
npx decap-server
```

Then reload `http://localhost:3000/admin`.

### Need a minimal CMS smoke test

Temporarily use `public/admin/config-simple.yml` as the active config only for diagnosis. Do not leave simplified config committed unless intentionally changing the maintained CMS mode.

### Production OAuth

Remote browser editing may require a GitHub OAuth proxy or another Decap-compatible auth backend. Historical Netlify Identity/Git Gateway notes were intentionally removed from the top-level docs because the maintained default is local editing.

If production CMS editing becomes necessary, document the selected auth path here before changing `public/admin/config.yml`.

## Safety

- Do not commit CMS OAuth client secrets.
- Do not commit private drafts unless they are meant to be public.
- Check generated Markdown frontmatter before committing.
- Run `npm run lint` and `npm run build` after structural CMS config changes.
